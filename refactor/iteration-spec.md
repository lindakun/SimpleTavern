## 一、P0 - 核心功能缺失

### 1.1 前端世界书（World Book）管理 UI
**需求描述**: 后端已完整实现世界书 CRUD（admin-worlds.controller + public-worlds.controller），但前端和 admin 面板均无世界书管理界面。用户无法通过 UI 创建/编辑/删除世界书条目。

**价值**: 世界书是角色扮演 AI 的核心上下文能力，缺失此功能严重影响产品核心竞争力。

**技术要点**:
- 前端新增 `WorldBookScreen.tsx` 组件
- 调用 `/api/worlds` 系列 API
- 支持世界书条目的增删改查、关键词触发配置
- 参考 SillyTavern 的世界书 UI 交互设计

**验收标准**:
- [ ] 用户可创建/编辑/删除世界书
- [ ] 支持 UID 标注、关键词配置、优先级设置
- [ ] 世界书可关联到角色

**预估工作量**: 3-4 天

---

### 1.2 密码找回前端完整流程
**需求描述**: 后端 `auth.service.ts` 已实现 `recover` / `recover-step2`，但前端 `LoginScreen` / `RegisterScreen` 无密码找回入口和流程页面。

**价值**: 基础用户管理能力，缺失会导致用户永久丢失账户。

**技术要点**:
- 新增 `ForgotPasswordScreen.tsx`（输入邮箱/用户名）
- 新增 `ResetPasswordScreen.tsx`（输入 reset token + 新密码）
- 连接后端 `/api/users/recover` 和 `/api/users/recover-step2`

**验收标准**:
- [ ] 登录页有"忘记密码"入口
- [ ] 用户可通过邮箱触发密码重置邮件
- [ ] 用户可通过 token 重置密码

**预估工作量**: 1-2 天

---

### 1.3 角色评论（Reviews）前端展示与交互
**需求描述**: 后端有 `reviews.repository.ts`，`characters.repository.ts` 中角色数据含 `reviews` 字段，前端 `CharacterDetailScreen` 未完整展示评论列表和提交 UI。

**价值**: 社交互动的核心功能，影响社区氛围和用户留存。

**技术要点**:
- `CharacterDetailScreen` 底部增加评论列表展示
- 新增评论提交表单（已部分实现 `handleAddReview`，需完善 UI）
- 支持评分展示（star rating）

**验收标准**:
- [ ] 角色详情页展示评论列表
- [ ] 登录用户可提交评论
- [ ] 评论支持评分（1-5 星）

**预估工作量**: 1-2 天

## 二、P1 - 高价值体验提升

### 2.1 聊天消息高级功能（重新生成、编辑、分支）
**需求描述**: 当前聊天功能仅支持发送和接收，缺少主流 AI 聊天应用必备的消息编辑、重新生成、分支对话能力。

**价值**: 显著提升用户与 AI 交互的掌控感，减少重复输入，是用户留存的关键功能。

**技术要点**:
- 消息气泡增加"重新生成"、"编辑"、"删除"按钮
- 编辑消息后截断后续对话并重新请求 AI
- 分支对话：允许从任意消息创建新分支
- 后端需支持 `history` 截断控制

**验收标准**:
- [ ] 用户可点击"重新生成"获取不同回复
- [ ] 用户可编辑已发送消息并重新提交
- [ ] 用户可删除单条消息
- [ ] 视觉上区分不同版本回复

**预估工作量**: 3-5 天

---

### 2.2 角色搜索与筛选（Discover 页面）
**需求描述**: Discover 页面目前展示全部角色，无搜索、标签筛选、排序功能。

**价值**: 角色数量增长后，用户无法快速找到感兴趣的角色，严重影响发现体验。

**技术要点**:
- 新增搜索框（按名称、描述搜索）
- 新增标签/分类筛选（从角色 `tags` 字段）
- 新增排序选项（热门/最新/评分）
- 后端新增 `/api/characters/search` 端点（支持分页）

**验收标准**:
- [ ] 用户可通过关键词搜索角色
- [ ] 用户可按标签筛选
- [ ] 用户可切换排序方式
- [ ] 搜索结果支持分页/无限滚动

**预估工作量**: 2-3 天

---

### 2.3 聊天历史持久化与多会话管理
**需求描述**: 当前每个角色只有一个聊天线程，用户无法创建多个独立会话（如"第一天"、"第二天"等不同场景）。

**价值**: 多会话管理是角色扮演应用的核心能力，用户需要为不同情境创建独立对话。

**技术要点**:
- 后端 `chats.repository.ts` 需支持多会话（当前按 `characterId` 单文件存储）
- 前端 `MessageCenterScreen` 增加会话列表（而非单一线程）
- 支持会话重命名、删除
- 数据存储格式从单文件升级为按会话 ID 分文件

**验收标准**:
- [ ] 用户可为同一角色创建多个聊天会话
- [ ] 用户可重命名、删除会话
- [ ] 会话列表按最后更新时间排序

**预估工作量**: 3-4 天

---

### 2.4 Admin 面板功能完善
**需求描述**: 当前 Admin 面板仅有基础 Dashboard、Users、Characters、Worlds 页面框架，缺乏实际管理操作能力。

**价值**: 产品上线后必需的内容审核、用户管理、系统配置能力。

**技术要点**:
- Characters 页面：审核/下架角色、编辑角色、查看角色详情
- Users 页面：禁用/启用用户、重置密码、分配管理员
- Worlds 页面：审核世界书、删除违规内容
- Dashboard：增加活跃用户、消息量、Token 消耗等统计图表

**验收标准**:
- [ ] 管理员可审核用户提交的角色
- [ ] 管理员可禁用/删除用户
- [ ] 管理员可查看系统统计数据

**预估工作量**: 4-5 天

## 三、P2 - 重要功能补充

### 3.1 多模态输入（图片上传 + 视觉理解）
**需求描述**: 后端 LLM 配置支持配置多模态模型，但前端聊天不支持图片上传和发送。

**价值**: 多模态是当下 LLM 应用的重要能力，支持图片输入显著提升用户体验。

**技术要点**:
- 聊天输入框增加图片上传按钮
- 图片预览、删除、压缩
- 消息类型从纯文本扩展为多模态（`{ type: 'image_url', image_url: { url } }`）
- 后端 `buildMessages` 需支持 `image_url` 类型内容

**验收标准**:
- [ ] 用户可在聊天中上传图片
- [ ] 图片随消息一起发送给 AI
- [ ] 支持常见图片格式（PNG/JPG/WebP）

**预估工作量**: 2-3 天

---

### 3.2 角色卡片导入/导出 UI 完善
**需求描述**: 后端支持 PNG 角色卡导入导出，前端 `CreateCharacterScreen` 有基础导入，但导出功能和批量导入未完成。

**价值**: SillyTavern 用户迁移的核心功能，降低用户切换成本。

**技术要点**:
- `MyCharactersScreen` 增加批量导入按钮（支持多选 PNG）
- 增加导出按钮（单角色 PNG 下载）
- 导入进度展示、失败重试

**验收标准**:
- [ ] 用户可批量导入 PNG 角色卡
- [ ] 用户可导出角色为 PNG 卡
- [ ] 导入失败有清晰错误提示

**预估工作量**: 2 天

---

### 3.3 用户设置页面功能完善
**需求描述**: `SettingsScreen.tsx` 已有 UI 框架（云端备份、自动播放、渲染质量），但部分功能未实际生效。

**价值**: 个性化设置是用户留存的重要因素。

**技术要点**:
- 确认后端 `/api/users/settings` 读写是否正常
- 实现"云端备份"开关的实际逻辑（自动同步聊天记录）
- 实现渲染质量设置的实际效果（控制动画/粒子效果）
- 新增"清除缓存"、""数据导出"等实用选项

**验收标准**:
- [ ] 所有设置开关均能正常保存和读取
- [ ] 设置能实际影响应用行为
- [ ] 增加清除缓存、导出数据功能

**预估工作量**: 1-2 天

---

### 3.4 消息中心（MessageCenter）UI 完善
**需求描述**: `MessageCenterScreen` 展示聊天线程列表，但缺少搜索、未读标记、批量操作等功能。

**价值**: 用户聊天记录增多后，消息中心是核心导航入口。

**技术要点**:
- 增加线程搜索（按角色名）
- 未读消息数角标
- 批量删除、批量置顶
- 空状态友好提示

**验收标准**:
- [ ] 用户可搜索聊天线程
- [ ] 未读消息有明确视觉提示
- [ ] 支持批量操作

**预估工作量**: 1-2 天

---

### 3.5 角色创建/编辑体验提升
**需求描述**: `CreateCharacterScreen` 支持基础角色创建，但缺少富文本编辑器、角色卡预览、模板功能。

**价值**: 降低角色创建门槛，提升 UGC 质量。

**技术要点**:
- 增加角色卡实时预览（手机样式预览卡片）
- 支持从模板创建（预设性格模板）
- 支持从现有角色复制（"分身"功能）
- 增加 AI 辅助生成角色描述（调用 LLM 生成 personality/description）

**验收标准**:
- [ ] 角色创建时有实时预览
- [ ] 支持模板创建
- [ ] 支持 AI 辅助生成角色描述

**预估工作量**: 3-4 天

## 五、P4 - 探索性功能（长期规划）

### 5.1 多用户协作（群组聊天）
**描述**: 参考 SillyTavern 的 Group Chat 功能，支持多个角色 + 一个用户在一个会话中交互。

**技术要点**:
- 后端 `chats.repository.ts` 需支持 group chat 存储格式
- 前端新增 `GroupChatScreen`
- 消息路由：用户消息广播给所有角色，角色之间也可交互

**预估工作量**: 5-8 天

---

### 5.2 角色市场/社区功能
**描述**: 用户可发布角色到公开市场，其他用户可收藏、评论、fork（分叉创作）。

**技术要点**:
- 后端增加角色可见性控制（private/public/forked）
- 增加角色 fork API（复制角色到自己的库）
- 前端 Discover 页面增加"社区"标签页

**预估工作量**: 4-6 天

---

### 5.3 TTS（文本转语音）
**描述**: AI 回复可自动朗读，提升沉浸感。

**技术要点**:
- 后端增加 TTS 提供商接入（OpenAI TTS、Edge TTS、本地 TTS）
- 前端增加语音播放控件
- 支持选择声音、调节语速

**预估工作量**: 3-4 天

---

### 5.4 稳定扩散集成（AI 绘图）
**描述**: 角色可在对话中生成场景图片，或根据用户描述生成角色头像。

**技术要点**:
- 后端增加 Stable Diffusion 提供商接入（参考 SillyTavern 的 `/api/sd/*` 端点）
- 前端聊天中增加图片生成请求和展示
- 支持 img2img、inpainting

**预估工作量**: 5-7 天

---

### 5.5 RAG（检索增强生成）
**描述**: 允许用户上传文档（PDF、TXT），作为角色知识库，在聊天时自动检索相关内容注入 prompt。

**技术要点**:
- 后端增加向量数据库集成（Qdrant 或本地向量索引）
- 文档解析（PDF → 文本）
- 语义检索 + prompt 注入

**预估工作量**: 6-10 天

## 八、附录：技术决策建议

### A. 数据存储升级建议
当前使用 `node-persist` + JSON 文件存储，随着用户量增长会出现性能瓶颈。建议：
- **短期**：保持文件存储，但增加写入锁、备份机制
- **中期**：迁移到 SQLite（轻量级，适合单机部署）
- **长期**：支持 PostgreSQL（多实例部署）

### B. 实时通信升级建议
当前使用 SSE（Server-Sent Events）实现流式回复，考虑升级为 WebSocket：
- 支持双向实时通信（当前 SSE 只支持服务端→客户端）
- 支持服务端主动推送（新消息通知、系统公告）
- 建议使用 `socket.io` 或原生 WebSocket

### C. 国际化（i18n）架构建议
产品有国际化潜力，建议早期引入 i18n 架构：
- 使用 `react-i18next`
- 语言文件按模块拆分
- 默认支持：中文（简体）、中文（繁体）、英文、日文
